# housing-qa

## Purpose

The public grounded housing Q&A chat: unauthenticated tenants ask questions, answers
are grounded in the tenant-FAQ corpus, generated by Claude (Haiku via SDK, or the
`claude` CLI as keyless fallback), and guarded against internal-language leakage.
Cost-bounded by design because the endpoint is public.

## Workflow encoded

1. Turnstile CAPTCHA + per-IP rate limit (20 req/10 min) + daily ceilings.
2. **Retrieval** — `buildContext()` against the tenant-FAQ corpus; the
   `tenant_public` retrieval policy excludes staff/statewide surfaces entirely.
3. **Generation** — Anthropic SDK (metered) when `ANTHROPIC_API_KEY` set, else CLI
   fallback (`HOUSING_QA_CLI_FALLBACK=1`; spawned with `--tools ""` and `--`
   injection-safe argument termination).
4. **Output guard** — `guardTenantAnswer()` regex-screens for internal vocabulary
   (routing/property/mode keys) and replaces the whole answer on a hit.
5. PII-safe spend logging (tokens + estimated USD).

## Data model

Stateless — in-memory daily counters on a UTC day window.

## API surface

`POST /api/housing-qa` (public) → `{question}` → `{answer}`.
Admin kill-switch: `POST /api/admin/housing-qa {enabled:false}` (`housing_qa:admin`,
system_admin) — instant, no redeploy.

## Compliance anchors

Scope containment is the compliance story: the public surface can only retrieve the
tenant corpus (the Jun-11 scope-fix work, PR #289/#290), and the output guard is the
backstop. Budget caps bound blast radius of a public LLM endpoint.

## Flags & env

`HOUSING_QA_ENABLED` (default ON; anything ≠ "false") · `HOUSING_QA_DAILY_MAX`
(default 500/day; 0 blocks) · `HOUSING_QA_DAILY_BUDGET_USD` (default $10, SDK path) ·
`HOUSING_QA_CLI_FALLBACK` · `ANTHROPIC_API_KEY` · `CLAUDE_CLI_PATH`.
